Client/Developer Team Project Report

This is a client report for our final project. I designed a website and hired two other students to build it. I also built a homepage for another student and child pages for a third student.

Client Report

Content and Images

Q: How did you decide what content and images to use for your site plan?

A: I had an idea for a website a few years ago after I did an engine swap on my car and I wanted to create a place where people could find information about that specific swap. I built a very crude version of it that was not very polished. For this assignment, I designed a much cleaner version of the website that I will share with the relevent people attempting this engine swap.

Communication with Developer

Q: How did you communicate with your developer? Were there any issues with communication?

A: For this site, I did not have very much communication with the developer. We talked about the site in class a few times and when he had fininsed the site, he texted me a link. I gave him some feedback and he implemented it. The lead developer did a great job on this project.

Developer Expectations

Q: Did your developer meet your expectations for your site?

A: Yes, the lead developer met my expectations and did an amazing job at designing the site. The junior developer was a bit difficult to work with, as he did not respond to texts or emails and did not complete the child pages.

Future Projects

Q: What will you do differently next time you hire a team to do a project for you?

A: If I were to do a project similar to this again, I would like to have more control over the decelopers I hire, as the junior dev was difficult to work with and not responsive to communications. I had a similar problems with the junior developer on the site I built and the client on the site I was a junior developer for.

Lead Developer Report

Team Management

Q: How did you manage your development team and communicate the tasks to your junior dev?

A: For my lead developer tasks, I wrote the HTML and CSS for the home page. I tried to use somewhat broad CSS so the junior developer could adpot the same stylesheet for his child pages. The junior developer ended up not looking at my home page or CSS and he wrote his own stylesheet so it does not match the home page. I communicated this clearly to him, but I guess there was a breakdown in understandng along the way.

Client Communication

Q: How did you communicate with your client? Were there any issues?

A: I was able to communicate pretty regularly with the client. We used both Microsoft Teams and regular texting. We mostly spoke in class about the changes I made to the website and potential changes he wanted implemented

Publishing

Q: How did you handle publishing your client's site and sharing it with the client?

A: I published the site using GitHub Pages and handled version control with Git. This allowed me to make changes and also allowed the junior developer to pull and push changes.

GitHub Experience

Q: How was your experience with managing your site with GitHub?

A: I feel that I was able to handle the GitHub aspects of the project smoothly, but Gideon did have some difficulties with syncing changes he made. I do have previous expirence using Git, GitHub, and Pages, so that helped.

Junior Developer Report

Team Communication

Q: How did you communicate with your team lead? Were there any issues?

A: The communication for this section of the project was severely lacking. The lead developer missed the first several classes after the project started and was slow to respond to emails. Eventually I was able to get in the GitHub repository, but there was little information about the client's site. He never showed up to class when I was there and did not respond to emails. We essentially built a wireframe and filled it with lorem.

Expectations

Q: Did you know what was expected of you for the site?

A: No, the expectations were not very clear to me simply because we had no site plan, only a wireframe and color palette. There was also little direction from the client.

Contribution

Q: Were you able to contribute to your site?

A: Yes, I was able to contribute by creating HTML and some CSS for the child pages. I made 3 child pages for the project, but since there was no content, I just added lorem text.

GitHub Issues

Q: Did you run into any issues with GitHub and adding your changes to the site?

A: I did not run into any GitHub issues with this section of the project. I opened a blank instance of Visual Studio Code and copied the repository link into the "Clone Repository" section. It worked for me perfectly.

General Experience and Reflections

Challenges

Q: What was the most challenging part of this project for you? Were you able to overcome it?

A: The most challenging part of this project was the collaboraation aspect. Writing the code that was needed did not take very long, but the lack of direction from the client in my junior developer role was a bit frustrating. We were able to get a wireframe and clone it, though the content was never provided.

Enjoyment

Q: What did you enjoy most about this project?

A: I enjoyed the opportunity to work on a project in a setting more closely resembling a real client project. It was fun to get a site plan from a client, implement it, and incorperate feedback. It was also a great opportunity to learn more about teamwork and how version control tools like Git and GitHub were able to help the project run more smoothly.

Improvements

Q: What would you do differently if you could do the project again?

A :This project was fun, but it was a bit overwelming to work on both sites at the same time, especially with the lack of clear information and direction from the client. If I were to change this project, I might like to work on one site at a time, and accelerate the pace at which we work through each section of the project. This would allow students to focus on one aspect at a time.

Confidence

Q: Do you feel more confident in working on real-world projects after this experience? Explain.

A: Yes, I feel more confident working on real-world projects after completing this project. I was able to see some of the sticking points and issues that could arise when working collaboratively on a web development project, and I was able to work to overcome those issues, like the lack of information from a client, in the best way possible.
